xtarterize publishes an **AI agent skill** at `skills/xtarterize/` that teaches coding agents (Claude Code, GitHub Copilot, Cline, etc.) how to use the CLI effectively.

## How the skill gets invoked

The skill's description acts as a trigger — when you ask an agent about anything related to xtarterize, it loads the skill and gains expert knowledge. Common trigger phrases:

| You say... | What the agent learns |
|------------|----------------------|
| "Set up linting for this project" | Runs `check --format json`, parses task statuses, applies `init` or `add lint/biome` |
| "Check our conformance status" | Runs `check --format json`, reports which tasks are `skip` vs `new`/`patch` |
| "Add TypeScript strict mode" | Loads the task catalog, runs `add ts/strict --format json` |
| "What would change if I init?" | Runs `diff --format json`, presents structured file diffs |
| "This project has config conflicts" | Identifies `conflict` status, presents options without auto-applying |
| "Run diagnostics on our setup" | Runs `doctor --format json`, reports any `fail` diagnostics |
| "I need to restore a backup" | Uses `restore` to recover from `.xtarterize/backups/` |

Without the skill, an agent might guess flags, parse terminal output, or miss tasks entirely. With the skill loaded, it uses `--format json`, parses structured output, follows anti-patterns, and loads reference files on demand.

## What the skill covers

The skill teaches agents:

- All 8 CLI commands and their flags
- JSON output shapes for every command (`check`, `diff`, `list`, `doctor`, etc.)
- Parsing patterns — which JSON fields to inspect to decide the next action
- The 4 task statuses (`new`, `patch`, `skip`, `conflict`) and how to handle each
- Agent workflows for initializing projects, adding tasks, diagnosing failures, and restoring backups
- Anti-patterns — common mistakes agents should avoid
- When to load detailed reference files (task catalog, full command reference)

## Skill structure

```
skills/xtarterize/
  SKILL.md                          # Entry point (~104 lines)
  references/
    commands.md                     # Full flag reference + JSON output shapes
    tasks.md                        # Complete task catalog with IDs
```

The skill uses **progressive disclosure**: SKILL.md covers core workflows and decision tables; reference files are loaded on demand when the agent needs detailed information (e.g., exact task IDs, full flag descriptions).

## Agent-first design

The skill is optimized for AI agent consumption:

- **JSON output by default** — all commands shown with `--format json`; exact JSON shapes documented
- **Decision tables** — "parse this field → to decide next action" patterns for every command
- **MANDATORY loading triggers** — reference files use explicit "MANDATORY — READ ENTIRE FILE" language
- **Anti-patterns** — clear NEVER lists with reasoning
