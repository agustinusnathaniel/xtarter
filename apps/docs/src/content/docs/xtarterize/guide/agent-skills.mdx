xtarterize includes published **AI agent skills** that teach coding agents (Claude Code, GitHub Copilot, Cline, etc.) how to use the CLI, parse its structured output, and follow best practices.

## What are agent skills?

Agent skills are markdown files that teach AI coding agents how to work with specific tools, frameworks, and libraries. When an agent loads a skill, it gains expert knowledge about:

- Available CLI commands and their flags
- How to parse JSON output for decision-making
- Common workflows and anti-patterns
- When to load detailed reference files

xtarterize ships two skills:

| Skill | Location | What it teaches |
|-------|----------|-----------------|
| `xtarterize` | `skills/xtarterize/` | All CLI commands, JSON output shapes, task catalog, agent workflows |
| `create-xtarter-app` | `skills/create-xtarter-app/` | Scaffolding commands, template selection, project creation workflows |

## How agents discover the skills

The `.claude-plugin/plugin.json` file in the repository root registers both skills for agent system discovery:

```json
{
  "name": "xtarter-skills",
  "skills": [
    "./skills/xtarterize",
    "./skills/create-xtarter-app"
  ]
}
```

Agent systems that support skill discovery (such as Claude Code) will automatically find and load these skills when working with the repository.

## Skill structure

Each skill follows a progressive disclosure pattern:

```
skills/<tool>/
  SKILL.md                          # Entry point (<150 lines)
  references/
    commands.md                     # Full flag reference + JSON output shapes
    tasks.md                        # Task catalog with IDs (xtarterize only)
    templates.md                    # Template details (create-xtarter-app only)
```

- **SKILL.md** contains the core workflows, decision tables, and anti-patterns — enough for most tasks
- **Reference files** are loaded on demand when the agent needs detailed information (e.g., exact task IDs, full flag descriptions)

## Agent-first design

The skills are designed specifically for AI agent consumption:

- **JSON output by default** — all xtarterize commands shown with `--format json`; the skill documents exact JSON shapes agents can parse
- **Decision tables** — "parse this field → to decide next action" patterns for every command
- **MANDATORY loading triggers** — reference files use explicit language at relevant workflow steps
- **Anti-patterns** — clear NEVER lists with reasoning, preventing common mistakes
