Latest updates and feature releases for xtarterize.

## v1.15.0

_June 29, 2026_

### CLI

- `restore --yes`: skip confirmation, restore latest backup
- `restore --quiet`: suppress verbose output
- `add --all`: apply all new and patch tasks without interaction
- `doctor --json`: now visible in command-level `--help`

### create-xtarter-app

- `--quiet`: suppress banners, progress output, and decorative text
- `--json`: output scaffold result as structured JSON
- `--no-color`: disable colorized output
- `--yes`: now defaults to `next-chakra` template when `-t` is omitted
- Migrated from hand-rolled `HELP_TEXT` to citty auto-generated help

### Docs

- Document new CLI flags in Starlight reference docs
- Update agent skills for accurate flag references
- Fix JSON output example to match actual output shape

## v1.14.3 â€” June 29, 2026

### Fixes

- **Git hooks now work on npm, yarn, and bun.** Hooks generated by `xtarterize add release/git-hooks` now use the correct command format for each package manager (via `nypm`).
- **engines.node raised to >=24.** Projects using Node 18â€“23 now receive a clear error from npm instead of a cryptic runtime failure.
- **Backup collision resolved.** Files with names that could collide in the backup directory (e.g., `a/b.js` and `a__b.js`) now use a collision-safe encoding scheme.
- **Backup path traversal protection.** `xtarterize undo` and `xtarterize restore` now reject restore operations targeting files outside the project root.
- **Vite plugin dry-run accuracy.** `xtarterize diff` output for Vite plugin tasks (vite-plugin-checker, rollup-plugin-visualizer) now matches what `xtarterize add` actually writes.
- **Detection cache improvement.** Config directory changes (`.github/`, `.vscode/`, `.changeset/`) are now tracked at the file level, ensuring stale project profiles are invalidated when files within these directories change.

## v1.14.2 â€” June 17, 2026

Edge-case hardening across the entire pipeline â€” no new features, but significantly improved stability for all commands and surfaces.

- **CLI (`init`, `sync`, `diff`, `add`, `doctor`, `restore`):** try/catch guards prevent crashes on individual task failures; `--skip`/`--only` no longer filters phantom empty-string values; `doctor` uses `Promise.allSettled` for resilient diagnostics when one check fails
- **Core (backup, cache, detect, apply):** robust error handling for atomic writes with temp file cleanup; schema validation guards against corrupted cache entries; fixed React Native + React co-detection returning `null`; skipped count now correctly tracks explicit skips from check phase
- **Tasks (package-scripts, git-hooks, factory):** fixed `this.getScripts` undefined crash in `packageScriptsTask`; `commitMsgHook` now accepts a package manager parameter instead of hardcoding pnpm; corrected `check()` return values for `conflict` vs `new` status detection
- **UI (diffs, JSON output, interactive menus):** merged multi-diff view preserves all diffs instead of dropping earlier ones; JSON `ok` field now reflects actual conformance state not intention; multiselect cancel properly aborts via `abortIfCancelled`
- **create-xtarter-app:** Windows-compatible path handling; fixed pnpm overrides crash; async `readdir` replaces sync `readdirSync`; improved project name sanitization; proper handling of explicit `false` CLI flag values
- **Documentation:** outdated content refreshed â€” Node.js minimum bumped to 24; missing `--timing` and `--format` flags documented; Rspack added to detected bundlers; task applicability fixed for renovate/lint-staged; CLI path corrected from `apps/cli` to `apps/xtarterize`

## v1.13 â€” May 17, 2026

<Aside type="note" title="Oxlint & Oxfmt support">
  When Vite+ is detected, xtarterize now configures oxlint/oxfmt (via `vp`) instead of Biome as the default linting/formatting stack. - Non-recommended lint rules mapped from ESLint configs: `eqeqeq`, `prefer-const`, `no-shadow`, `max-params`, React rules, Vitest overrides for Oxlint; `noExcessiveCognitiveComplexity`, `useMaxParams` for Biome
- Generated config templates (Biome, Oxlint, Oxfmt, commitlint) now ship with proper TypeScript types
